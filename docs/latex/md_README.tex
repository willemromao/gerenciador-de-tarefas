\hypertarget{md_README_autotoc_md1}{}\doxysubsection{Descrição do Projeto}\label{md_README_autotoc_md1}
O projeto consiste em um gerenciador de tarefas implementado em C++, dividido em três arquivos principais\+: {\ttfamily \mbox{\hyperlink{Tarefa_8h}{Tarefa.\+h}}}, {\ttfamily \mbox{\hyperlink{Tarefa_8cpp}{Tarefa.\+cpp}}} e {\ttfamily \mbox{\hyperlink{GerenciadorTarefas_8cpp}{Gerenciador\+Tarefas.\+cpp}}}. O código é estruturado de forma a demonstrar conceitos importantes de programação, como alocação de memória dinâmica, manipulação de arquivos, construtores, destrutores, uso de memória estática, sobrecarga de função e herança.\hypertarget{md_README_autotoc_md2}{}\doxysubsection{Tarefa.\+h}\label{md_README_autotoc_md2}
Este arquivo contém a definição da classe {\ttfamily \mbox{\hyperlink{classTarefa}{Tarefa}}} e a classe derivada {\ttfamily \mbox{\hyperlink{classTarefaAmanha}{Tarefa\+Amanha}}}. A classe {\ttfamily \mbox{\hyperlink{classTarefa}{Tarefa}}} possui membros privados, como o identificador ({\ttfamily id}), descrição ({\ttfamily descricao}) e estado de conclusão ({\ttfamily concluida}). Há também um membro estático ({\ttfamily total\+Tarefas}) que mantém o total de tarefas criadas.

A classe {\ttfamily \mbox{\hyperlink{classTarefa}{Tarefa}}} possui métodos para obter o ID, a descrição, verificar se a tarefa está concluída, concluir a tarefa, exibir informações e métodos estáticos para obter o total de tarefas e atualizar esse total. A sobrecarga do operador de saída ({\ttfamily $<$$<$}) é utilizada para facilitar a exibição das informações da tarefa.

A classe derivada {\ttfamily \mbox{\hyperlink{classTarefaAmanha}{Tarefa\+Amanha}}} herda da classe {\ttfamily \mbox{\hyperlink{classTarefa}{Tarefa}}} e adiciona seu próprio membro estático ({\ttfamily total\+Tarefas\+Amanha}). Além disso, ela sobrescreve o método {\ttfamily exibir} para incluir informações específicas sobre tarefas programadas para o dia seguinte.\hypertarget{md_README_autotoc_md3}{}\doxysubsection{Tarefa.\+cpp}\label{md_README_autotoc_md3}
Neste arquivo, são implementados os métodos declarados na classe {\ttfamily \mbox{\hyperlink{classTarefa}{Tarefa}}} e {\ttfamily \mbox{\hyperlink{classTarefaAmanha}{Tarefa\+Amanha}}}. A alocação dinâmica de memória é utilizada para criar novas instâncias de tarefas. O destrutor é responsável por liberar a memória alocada para cada tarefa.

Os métodos de manipulação de tarefas, como adicionar, concluir e exibir, são implementados para facilitar a interação com o gerenciador. O arquivo também contém a implementação de funções estáticas para atualizar o total de tarefas.\hypertarget{md_README_autotoc_md4}{}\doxysubsection{Gerenciador\+Tarefas.\+cpp}\label{md_README_autotoc_md4}
O arquivo principal do programa implementa as funções de interação com o usuário, utilizando um menu simples. As funções incluem adição de tarefas, listagem, conclusão de tarefas, adição de tarefas para o dia seguinte, listagem de tarefas para o dia seguinte, salvamento das tarefas em um arquivo e desalocação de memória.

É importante observar que a desalocação de memória é realizada no final do programa para evitar vazamento de memória. Além disso, as funções de salvar tarefas em um arquivo e listar tarefas para o dia seguinte demonstram o uso de manipulação de arquivos.\hypertarget{md_README_autotoc_md5}{}\doxysubsection{Conclusão}\label{md_README_autotoc_md5}
O projeto fornece uma implementação prática de diversos conceitos de programação em C++, demonstrando a aplicação de boas práticas, herança, manipulação de memória, interação com o usuário e persistência de dados. O código está estruturado de forma modular, facilitando a compreensão e manutenção do sistema.\hypertarget{md_README_autotoc_md6}{}\doxysubsection{Compilação}\label{md_README_autotoc_md6}
